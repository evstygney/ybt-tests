# HOW_TO_EDIT

## Для редактора контента без кода

1. Откройте `/admin`.
2. В таблице модулей включите или выключите нужное упражнение, задайте порядок и неделю курса.
3. Перейдите в `Редактировать`.
4. Меняйте стандартные поля формы:
   - название;
   - цель;
   - неделя;
   - бейдж;
   - slug;
   - рекомендация на 48 часов.
5. Нажмите `Сохранить`. Изменения сохраняются в браузере.
6. Для предпросмотра используйте правую панель `Preview`.
7. Для передачи конфига нажмите `Экспорт JSON`.

## Если нужно продвинутое редактирование

1. Нажмите `Показать JSON`.
2. Отредактируйте JSON вручную.
3. Нажмите `Применить JSON`.
4. Если JSON невалиден, редактор покажет ошибку.

## Импорт нового конфига

1. Включите режим `Показать JSON`.
2. Нажмите `Импорт JSON`.
3. Выберите файл.
4. Проверьте preview.
5. Нажмите `Применить JSON` и затем `Сохранить`.

## Как публиковать упражнение отдельно

- У каждого упражнения есть собственный URL: `/exercise/[slug]`.
- Для LMS/iframe используйте `/exercise/[slug]?embed=1`.
- Если вы меняете `slug`, ссылка на упражнение тоже меняется.
- Для GitHub Pages новый `slug` должен существовать в исходных файлах проекта на момент сборки. Изменение slug только в локальной админке не создаёт новую страницу на опубликованном сайте.

## Если сайт лежит на GitHub Pages

- Базовые 12 упражнений публикуются автоматически.
- Все изменения в JSON-файлах проекта публикуются после нового workflow deploy.
- Локальные данные админки, прогресс и аналитика остаются в браузере конкретного пользователя.
